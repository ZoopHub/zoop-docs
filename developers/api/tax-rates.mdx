Tax rates are tenant-level configuration records. Each rate has a name and a percentage value. One rate can be marked as the **tenant default** — the rate Zoop applies automatically when creating new invoices, unless the caller overrides it.

All tools in this group require `read:tax_rates` or `write:tax_rates`. There is no umbrella scope that expands to cover these — the pair stands alone. See [Scopes](/developers/authentication/scopes) for the full catalog.

<Callout type="info">
If you are new to the MCP layer, start at [MCP overview](/developers/mcp) to learn how to connect and authenticate.
</Callout>

---

## Scopes

| Scope | Grants access to |
|---|---|
| `read:tax_rates` | `tax_rates.list`, `tax_rates.get` |
| `write:tax_rates` | `tax_rates.create`, `tax_rates.update`, `tax_rates.archive` |

`write:tax_rates` does **not** imply `read:tax_rates`. Grant both if your integration needs to read and write.

Unlike many other write tools, the tax rate tools **do not require a user actor**. Tenant-key tokens (no user ID) can call all five tools, because tax rates have no `created_by` attribution column.

---

## How rates are stored

Tax rates are stored internally as a decimal — for example, `0.0825` represents 8.25%. This is the value you will see in `get` and `list` responses in the `rate_decimal` field.

When **creating or updating** a rate, supply the human-readable percentage in `rate_percentage` (e.g. `8.25`). Zoop converts it to the stored decimal. You never write `rate_decimal` directly — attempting to send it to `tax_rates.update` is an `invalid_input` error.

Valid range: `0` to `99.9999`, with at most 4 decimal places. A value of `99.99999` or higher would round to a stored decimal of `1.0`, which is not allowed.

## Tools

### `tax_rates.list`

List all active tax rates for the tenant, most recent first. The response also includes `default_tax_rate_id` — the `id` of whichever rate is currently the tenant default, or `null` if no default is set.

Archived rates are excluded by default. Pass `include_archived: true` to include them.

<ParamField body="include_archived" type="boolean" optional>
  When `true`, returns archived (soft-deleted) rates alongside active ones. Defaults to `false`.
</ParamField>

#### Example

```json
{
  "jsonrpc": "2.0",
  "id": 1,
  "method": "tools/call",
  "params": {
    "name": "tax_rates.list",
    "arguments": {}
  }
}
```

```json
{
  "jsonrpc": "2.0",
  "id": 1,
  "result": {
    "content": [
      {
        "type": "text",
        "text": "{\"rates\":[{\"id\":\"d4e5f6a7-...\",\"name\":\"CA sales tax\",\"rate_decimal\":0.0825,\"description\":null,\"is_active\":true,\"created_at\":\"2026-01-10T09:00:00.000Z\"}],\"default_tax_rate_id\":\"d4e5f6a7-...\"}"
      }
    ]
  }
}
```

### `tax_rates.get`

Fetch a single tax rate by `id`. Returns `not_found` if the rate does not exist in the tenant or has been archived.

<ParamField body="id" type="string (UUID)" required>
  The tax rate's `id`. Get this from `tax_rates.list`.
</ParamField>

---

### `tax_rates.create`

Create a new tax rate. The name must be unique among active rates in the tenant — a duplicate name is a `conflict` error.

<ParamField body="name" type="string" required>
  Display name for the rate. Max 60 characters. Must be unique among active rates.
</ParamField>

<ParamField body="rate_percentage" type="number" required>
  The tax rate as a human percentage: `8.25` means 8.25%. Stored internally as `0.0825`. Must be between `0` and `99.9999` with at most 4 decimal places.
</ParamField>

<ParamField body="description" type="string" optional>
  Optional description or notes about this rate. Max 500 characters. Pass `null` to store an explicit empty value.
</ParamField>

<ParamField body="is_default" type="boolean" optional>
  Set to `true` to make this the tenant default. If another rate is already the default, it is automatically demoted. Defaults to `false`.
</ParamField>

#### Example — create a rate and set it as the default

```json
{
  "jsonrpc": "2.0",
  "id": 2,
  "method": "tools/call",
  "params": {
    "name": "tax_rates.create",
    "arguments": {
      "name": "CA sales tax",
      "rate_percentage": 8.25,
      "description": "California statewide base rate",
      "is_default": true
    }
  }
}
```

```json
{
  "jsonrpc": "2.0",
  "id": 2,
  "result": {
    "content": [
      {
        "type": "text",
        "text": "{\"id\":\"d4e5f6a7-...\",\"name\":\"CA sales tax\",\"rate_decimal\":0.0825,\"description\":\"California statewide base rate\",\"is_active\":true,\"created_at\":\"2026-06-13T18:00:00.000Z\"}"
      }
    ]
  }
}
```

### `tax_rates.update`

Update a tax rate's mutable fields. This is a partial patch — only the fields you send are changed. Fields you omit stay as they are.

You cannot patch `rate_decimal` directly (use `rate_percentage` instead) or `is_active` (use `tax_rates.archive` to deactivate a rate). Attempting either returns `invalid_input`.

<ParamField body="id" type="string (UUID)" required>
  The tax rate to update.
</ParamField>

<ParamField body="name" type="string" optional>
  New name. Max 60 characters. Must remain unique among active rates — a name already in use by another rate is a `conflict` error.
</ParamField>

<ParamField body="rate_percentage" type="number" optional>
  New rate as a human percentage (e.g. `9.5`). Same constraints as `tax_rates.create`.
</ParamField>

<ParamField body="description" type="string" optional>
  Updated description. Max 500 characters. Pass `null` to clear.
</ParamField>

<ParamField body="is_default" type="boolean" optional>
  Pass `true` to make this the tenant default. Pass `false` to clear the default if this rate is currently the default (the default becomes `null`). Has no effect if `false` and this rate is not currently the default.
</ParamField>

#### Example — change the rate percentage

```json
{
  "jsonrpc": "2.0",
  "id": 3,
  "method": "tools/call",
  "params": {
    "name": "tax_rates.update",
    "arguments": {
      "id": "d4e5f6a7-...",
      "rate_percentage": 9.0
    }
  }
}
```

---

### `tax_rates.archive`

Soft-archive a tax rate. Archived rates are hidden from default list results but the record is preserved — any invoices or quotes that used it retain their correct totals.

If the rate being archived is the tenant default, **the default is automatically cleared** (set to `null`). Zoop will not apply any tax rate automatically until you set a new default.

Archiving a rate that is already archived (or does not exist) returns `not_found`. There is no restore tool in the current MCP version.

<ParamField body="id" type="string (UUID)" required>
  The tax rate to archive.
</ParamField>

<Callout type="warning">
Archiving the default rate clears the tenant default. New invoices and quotes will have no tax rate pre-applied until you set a new default with `tax_rates.update` or `tax_rates.create`.
</Callout>

#### Example

```json
{
  "jsonrpc": "2.0",
  "id": 4,
  "method": "tools/call",
  "params": {
    "name": "tax_rates.archive",
    "arguments": {
      "id": "d4e5f6a7-..."
    }
  }
}
```

```json
{
  "jsonrpc": "2.0",
  "id": 4,
  "result": {
    "content": [
      {
        "type": "text",
        "text": "{\"archived\":true,\"id\":\"d4e5f6a7-...\"}"
      }
    ]
  }
}
```

## Managing the tenant default

The tenant default is a single pointer — at most one rate is marked as the default at any time. Here is the typical lifecycle:

<Steps>
  <Step title="Create the rate">
    Call `tax_rates.create` with `is_default: true`. If a default already exists, it is automatically demoted.
  </Step>
  <Step title="Change the default">
    Call `tax_rates.update` on the new rate with `is_default: true`. The previous default is demoted automatically.
  </Step>
  <Step title="Clear the default">
    Call `tax_rates.update` on the current default with `is_default: false`. The default becomes `null`. No tax rate will be pre-applied to new documents until you set another.
  </Step>
  <Step title="Archive the default rate">
    Call `tax_rates.archive`. The rate is hidden and the default is cleared automatically — same outcome as clearing it explicitly.
  </Step>
</Steps>

To read the current default at any time, call `tax_rates.list` and read `default_tax_rate_id` from the response.

## Error reference

| Kind | What it means | Common cause |
|---|---|---|
| `invalid_input` | A field failed validation | Missing `name` or `rate_percentage`, `rate_percentage` out of range or too many decimal places, attempting to patch `rate_decimal` or `is_active` directly |
| `not_found` | The rate does not exist in this tenant or is already archived | Stale `id`, rate was archived before this call |
| `conflict` | The name is already in use by another active rate | Duplicate name on `create` or `update` |
| `insufficient_scope` | The token does not hold the required scope | Missing `read:tax_rates` or `write:tax_rates` |

For the full error shape and HTTP status codes, see [Errors](/developers/errors).
